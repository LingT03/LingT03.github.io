# Ling.\_.T — Personal Portfolio

A personal portfolio site built to the specification in
`Portfolio_Website_Guideline.pdf`. The frontend is React + Vite +
TypeScript + Tailwind CSS + Framer Motion. Content lives as Markdown
with YAML front matter and is compiled into typed JSON by a Python
build pipeline with Pydantic validation.

## Architecture

```
.
├── content/                  # source of truth for all displayed data
│   ├── degrees/*.md          # Degree records
│   ├── courses/*.md          # Course rows tied to a degree_id
│   ├── jobs/*.md             # Job records
│   ├── projects/*.md         # Project records
│   ├── books/*.md            # Book reviews (managed by fetch_books.py)
│   └── _meta/
│       ├── tech_stack.json   # shared tech badges
│       └── devices.json      # capture devices for the Hobbies page
├── scripts/                  # Python build pipeline (runs in venv)
│   ├── schemas.py            # Pydantic source of truth
│   ├── build_content.py      # content → /public/data/*.json
│   ├── fetch_books.py        # Open Library ingestion (V2 §3.5.1)
│   └── requirements.txt
├── frontend/                 # React + Vite app
│   ├── src/{components,pages,lib,lib/hooks}/
│   └── public/data/          # generated by scripts/build_content.py
└── .github/workflows/deploy.yml
```

`content/photos/` is an optional expansion hook — the `Photo` schema is
wired through the pipeline, the `Hobbies` page reads any photos it
finds, and the build emits an empty `photos.json` when the directory is
absent.

## Local development

The Python pipeline runs in a `venv` (Python 3). The React app runs
npm scripts that invoke the pipeline before `vite dev` / `vite build`.

```bash
# 1. Python venv (run once)
python3 -m venv .venv
source .venv/bin/activate
pip install -r scripts/requirements.txt

# 2. Compile content → JSON (or rely on the `predev` hook)
python scripts/build_content.py

# 3. (Optional) refresh book metadata from Open Library
python scripts/fetch_books.py            # online: fetch + merge
python scripts/fetch_books.py --offline  # merge catalog only

# 4. Start the frontend dev server
cd frontend
npm install
npm run dev
```

Visit http://localhost:5173.

`npm run dev` and `npm run build` both run `python scripts/build_content.py`
first via the `predev` / `prebuild` hooks (see `frontend/package.json`).

## Adding content

Edit or add Markdown files under `content/`. Every record's front
matter is validated against a Pydantic schema in `scripts/schemas.py` —
unknown fields are forbidden (`extra="forbid"`) and the build fails
with a precise error message on the first invalid record.

Example: add a new book.

```markdown
---
id: book-new-id
title: Some Title
author: Author Name
category: Nonfiction # one of: Fiction | Nonfiction | Textbook
rating: 8.5 # 0.00–10.00, optional for textbooks
cover_image_url: "https://covers.openlibrary.org/b/id/.../-L.jpg"
open_library_key: "/works/OL12345W"
isbn: "9780000000000"
started_at: 2025-01-12
finished_at: 2025-03-04
tags: [statistics, methods]
---

A short summary in your own voice.
```

Then either restart `npm run dev` or rerun `python scripts/build_content.py`.

Book covers, authors, and `open_library_key` values can also be
populated automatically by running `python scripts/fetch_books.py` — it
merges canonical bibliographic fields fetched from Open Library with
your locally authored review fields (rating, tags, summary, notes).

## Routing

| Route           | Page component           | Notes                                                            |
| --------------- | ------------------------ | ---------------------------------------------------------------- |
| `/`             | `pages/Home.tsx`         | Split-pane dashboard (§4.2)                                      |
| `/academic`     | `pages/Academic.tsx`     | Degree timeline, course tooltips, undergrad transcript flowchart |
| `/work`         | `pages/Work.tsx`         | V2 §3.3 — formerly `/professional`                               |
| `/projects`     | `pages/Projects.tsx`     | Card grid + modal with blurred backdrop                          |
| `/books`        | `pages/Books.tsx`        | Fiction / Nonfiction / Textbook filter chips (V2 §3.5)           |
| `/hobbies`      | `pages/Hobbies.tsx`      | V2 §3.6 — formerly `/photography`                                |
| `/certificates` | `pages/Certificates.tsx` | Skeleton view; catalog under construction                        |
| `/api-docs`     | `pages/ApiDocs.tsx`      | Live backend + schema reference                                  |
| `*`             | `pages/NotFound.tsx`     | 404                                                              |

`/professional`, `/profiles`, and `/photography` redirect to their V2
equivalents via `<Navigate replace>` so external links keep resolving.
